Android Intro
Beginner's introduction to Android

General Information about Android

  • Named after creator Andy Rubin, versions named after desserts
  • Android Open Source Project (AOSP), forks: CyanogenMod, Amazon Kindle Fire, Nokia X
  • Linux kernel, Apps written in Java 6
  • Supports HTML5 (e.g. Apache Cordova)
  • Tools are free, available on all platforms (PC/Mac/Linux)

General Information about Android

  • Google Play: $25 signup fee, no annual fee, 70% revenue split (ZA not supported)
  • ZA Options: Ads, Amazon/Samsung/Vodacom app stores, offshore account
  • Market share: 79% of smartphones shipped worldwide in 2013 (Strategy Analytics)

Getting the tools

  • http://developer.android.com
  • Download the ADT Bundle (~450Mb). Android Studio for IntelliJ fans.

Let's start!

  • File -> New -> Other -> Android -> Android Application Project

Run your app!

  • Create an emulator:
    • Android 4.x, HVGA, Intel image (if avail)
    • Windows/Mac: Install Intel HAXM driver from Intel site
    • Genymotion.com - fast emulator, uses VirtualBox and RAM/CPU
  • Right-click project root -> Run As -> Android application

Run your app!

  • Use your device:
    • Settings -> About -> Tap Build Number many times
    • Settings -> Developer Options -> Enable USB Debugging
    • Plug in your device via USB (Windows needs USB driver)
    • Device must be visible in Devices view
  • Right-click project root -> Run As -> Android application

Main components of an App

  • Activities - various screens in your app (like web pages)
    • Fragments - like iframes, stand-alone UI components that run inside an Activity, can be dynamically added/removed
  • Services - long-running tasks in your app (e.g. downloading file)
  • Broadcast Receiver - listens for broadcast intents (e.g. phone ringing, battery low)

Android project folders

  • src – All Java code
  • assets – multimedia, data, etc bundled with app
  • res – all user interface resources
    • drawable – graphics and drawable xml
    • layout – all activity layouts
    • values – strings, colours, styles, etc.
    • menu - menu XML descriptors
  • bin, gen - generated output, ignore in version control
  • AndroidManifest.xml – app descriptor file

Android Manifest file

  • Declares application, activities, services, permissions, hardware required, etc
  • Used by Market to filter apps based on SDK and hardware features
  • Used by installer to create launcher icon
  • Intent filters tell Android what operations your app can handle (e.g. E-mail, dialer, websites)

Connecting the dots

  • How does Android know which class and layout to load?
  • On install, creates icons for Intent with action MAIN and category LAUNCHER
  • On launch, it calls onCreate() method of the specified class, followed by onStart() and then onResume()
  • The rest is up to the class!
  • In onCreate() we use setContentView() to load the layout
  • The generated R class allows code to reference the resources in the res folder

Button and Toast

  • Delete the layout content and add a LinearLayout (vertical)
  • Add a Button to the layout
  • Add an onClick handler to the Activity
  • In the onClick, display a Toast

Exceptions and LogCat

  • "Force Close" (FC) - uncaught Java Exception
  • "Application Not Responsing" (ANR) - UI thread hanging
  • Use the android.util.Log class to log output
  • Open the LogCat view to view the logs and filter them

The all-important Action Bar

Dialog boxes

  • AlertDialog.Builder lets you create common dialog boxes
  • Theme.Dialog allows you to change an Activity into a Dialog

Layouts

  • LinearLayout - stacks views horizontally or vertically
  • FrameLayout - stacks views over each other
  • RelativeLayout - allows relative positioning of views
  • TableLayout - puts views into a table, e.g. forms
  • Views can be positioned using margin, padding, visibility, gravity, layout weight
  • Note: Android uses dips (dp) - density-independent pixels

Drawables

  • Shape - rectangle, oval, line, ring with fill, stroke.
  • Selector - higlights touched/selected/active/disabled elements
  • Bitmap - image that can be tiled in many ways, e.g. pin-stripes
  • 9-patch - stretchable PNG
  • Drawables can be displayed using an ImageView or set as background for most layouts and widgets

Stuff you need to know

  • Handling configuration changes (e.g. rotation)
  • List Adapter to populate a ListView/GridView/Gallery
  • Multi-threading with AsyncTask or IntentService
  • Efficient networking - use the Volley library
  • Preferences and PreferenceActivity for displaying/saving settings
  • Adapting to different screen sizes and orientations with Fragments, SlidingPane, and NavigationDrawer
